Фабрика
Инкапсуляция создания объектов:
1. любые сущности на сцене - кнопочки, мобы, тайлы, части карт и тд
2. классы, которые требуют множества зависимостей (например какие нибудь специфические контроллеры, для других сущнсотей)

Визитёр (тут забыл, буду писать как понял)
Используется для разделения ответственности - мы выносим из классов какой либо общий функционал не относящийся напрямую к этому классу.
А так же, что бы понять логику для какого типа мы должны в текущий момент выполнить.
Пример из урока с подсчётом очков. Думаю можно использовать для воспроизведения звука на основе типа.
Или выполнять другую более сложную логику.

Декоратор
Используется для изменения текущего поведения объекта с помощью его вложения в другой объект, который имеет тот же интерфейс.
Например, мы можем задекорировать кофе молоком и будет кофе с молоком по суммарной цене кофе и молока =)
Или пример из урока - с расчётом ХП для разных рас в игре.
Так же можно поменять расчёт урона в зависимости от бафов-дебафов для каждой расы и тд.
